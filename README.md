天体観測支援システム：NEXUS [ZENITH:ZERO] 開発要求定義書
1. 開発の目的 (Purpose)
NEXUS [ZENITH:ZERO] は、**「今、この場所の空の真の状態を正確に診断し、ユーザーが保有する機材で最大限の観測成果を引き出す」**ための天体観測支援システムである。

スマホのカメラとセンサーを活用した高精度なリアルタイム環境解析を基盤とし、SQM、NELM、光害指数、月影響度などのクリティカルデータを即座に可視化・数値化する。

これにより、初心者には確実な成功体験を、中級者には最適な機材選択を、上級者には露出・ゲイン・スタック戦略の論理的根拠を提供し、無駄な試行錯誤を最小限に抑え、観測効率と成果を最大化することを目的とする。

2. ターゲットユーザーと提供価値 (User Value)
2.1 初心者（ビギナー層）
課題: 何を見ればいいのか、どう始めればいいのか分からない。

提供価値: 現在の空の状態（SQM/Bortle/NELM）に基づき、手持ち機材で確実に視認・撮影可能な天体を自動推奨。ARナビゲーションにより、最短距離で「成功体験」を得られる。

2.2 中級者（ステップアップ層）
課題: 見たい天体に対して、どの機材の組み合わせがベストか判断に迷う。

提供価値: **【機材逆引き提案機能】**により、目標天体に対して所持している鏡筒・カメラ・アイピースの最適な組み合わせ（焦点距離、F値、画角、明るさ）をシミュレーションして提示。

2.3 上級者（ガチ勢・EAA層）
課題: 露出時間、ゲイン、スタック枚数、フィルター選択の根拠となる信頼できる数値が欲しい。

提供価値: 高精度なSQM、NELM、光害指数、月齢ペナルティなどの生データをリアルタイムで提供。電子観望（EAA）における戦略立案の科学的根拠とする。

3. 主要機能要件 (Functional Requirements)
3.1 環境解析エンジン (NEXUS-CORE)
スマホカメラを用いたリアルタイムSQM測定（ダークフレーム補正、マルチフレームノイズ除去を含む）。

Persistent Star（持続性星点）検出による空の透明度評価。

GPS連動による月齢・月高度ペナルティの自動計算。

光害指数（R/B比ベースの波長解析）の算出。

3.2 機材データベース & シミュレーター
ユーザー所持機材（鏡筒、カメラ、アイピース、バローレンズ等）の登録機能。

順引き機能: 現在の空の状態 ＋ 所持機材 → 観測可能天体リストの生成。

逆引き機能: 目標天体 → 最適機材組み合わせの提案（焦点距離、F値、画角、予想像の明るさ）。

3.3 拡張Astro-HUD
AR画面上に以下のクリティカル情報を常時表示：

SQM / Bortle: 夜空の明るさ。

NELM: 肉眼限界等級。

Stars Count: 安定して検出された星の数。

Pollution Index + Moon Penalty: 光害と月の影響度。

Orientation: 天頂からのオフセット角（キャリブレーション精度）。

3.4 データポータビリティ
測定結果を window.NEXUS_DATA として構造化。

外部モジュール（Astro Pro、Analyzer等）から容易に参照可能なAPI設計。

4. 非機能要件 (Non-Functional Requirements)
暗順応配慮: 赤色基調の「NIGHT MODE（Tactical IR）」を標準搭載。

操作性: 天頂ガイダンスの明確化、プログレスバーの視認性向上、Matrix Rainの情報設計最適化。

拡張性: コードは常に完全形で管理し、将来的なモジュール連携を前提とした設計とする。

パフォーマンス: スマホ単体でリアルタイム動作可能（低解像度サンプリングを活用）。

5. システム構成（将来展望）
本システムを以下の3つの役割で位置づける：

NEXUS (本機): 空を「診る」高精度センサー・計器（感覚器官）。

Astro Pro: 観測計画・スケジュール立案の脳（思考機関）。

Analyzer: 撮影結果と環境データを紐づけ、次回観測の最適化を行う（記録・分析系）。
